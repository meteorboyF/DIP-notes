Introduction to Digital Images

From analog signals to discrete pixel arrays

1.1 What is a Digital Image?

An image is represented as a two-dimensional function f(x, y):

$$f(x,\,y)$$

x and y are the spatial (plane) coordinates  Г‚В·  f is the amplitude (intensity) at (x, y)

  • Г°ЕёвЂњВЌ x and y are the spatial (plane) coordinates.
  • Г°ЕёвЂњВђ The amplitude of f at any pair of coordinates (x, y) is called the intensity or gray level of the image at that point.
  • Г°ЕёвЂќВў It is composed of a finite number of elements called pixels (picture elements), each of which has a particular location and value.

Digital Image Definition: If x, y, and the amplitude values of f are all finite and discrete quantities, we call the image a digital image.

Г°ЕёвЂ“ВҐГЇВёВЏ Image as a Matrix

In computers, images are represented as matrices of numbers organized in rows and columns. This is the syntax for a digital image of size M ГѓвЂ” N:

$$f(x,y) = \begin{bmatrix} f(0,0) & f(0,1) & \cdots & f(0,N{-}1) \\ f(1,0) & f(1,1) & \cdots & f(1,N{-}1) \\ \vdots & \vdots & \ddots & \vdots \\ f(M{-}1,0) & f(M{-}1,1) & \cdots & f(M{-}1,N{-}1) \end{bmatrix}$$

M = rows, N = columns  Г‚В·  x ГўЛ†Л† [0, MГўЛ†вЂ™1], y ГўЛ†Л† [0, NГўЛ†вЂ™1]

Г°ЕёвЂќВІ Rows & Columns

The matrix is composed of M rows and N columns.

Г°ЕёвЂќВµ Each element = pixel

Every element in this matrix is a picture element or pixel.

Г°ЕёвЂњЕ  8-bit value range

In an 8-bit image, every matrix position holds a value from 0 to 255.

Sampling, Quantization & Resolution

The mathematics of continuous-to-discrete conversion

2.1 Spatial Resolution

Spatial resolution defines how many pixels per unit area. It is measured in DPI (Dots Per Inch) or PPI (Pixels Per Inch). Higher DPI = more pixels = more detail captured.

$$\text{Total Pixels} = M \times N$$ $$\text{Spatial Resolution (DPI)} = \frac{\text{pixels}}{\text{inch}}$$

Effect of Reducing Spatial Resolution

  • ГўВ¬вЂЎГЇВёВЏ Fewer pixels ГўвЂ вЂ™ Jagged lines (staircase effect)
  • ГўВ¬вЂЎГЇВёВЏ Fine details (tiny tumors) become invisible
  • ГўВ¬вЂЎГЇВёВЏ Objects lose shape fidelity
  • ГўЕ“вЂ¦ Smaller file size

Jagged Lines (Aliasing)

When spatial resolution is too low, smooth diagonal edges appear as a "staircase" pattern because a coarser grid cannot represent fine angular detail.

2.2 Intensity Resolution

Intensity resolution defines how many distinct gray levels can be represented per pixel. It is determined by the number of bits per pixel (bit depth).

$$L = 2^k \quad \Rightarrow \quad \text{gray levels from } 0 \text{ to } L-1$$ $$\text{Intensity range: } [0,\ 2^k - 1]$$

ГўЕЎВ ГЇВёВЏ False Contouring

When intensity resolution is too low (e.g., 2-bit = 4 levels), smooth gradients appear as abrupt bands of constant color Гўв‚¬вЂќ called false contours. This happens because nearby tones that should be slightly different are mapped to the same discrete level.

Example: An X-ray with only 3 bits (8 levels) cannot distinguish subtle density changes in tissue, leading to incorrect diagnosis due to loss of tissue contrast.

Bits/pixelGray LevelsTissue ContrastProblem
3-bit 8 Very Poor False contouring, missed soft tissue differences
8-bit 256 Good Acceptable for most uses
12-bit 4096 Excellent Ideal for medical imaging (MRI, CT)

2.3 Spatial vs. Intensity Resolution Tradeoff

Prototype 1 High Spatial, Low Intensity
  • ГўЕ“вЂ¦ Captures fine spatial detail (edges, small structures)
  • ГўЕ“вЂ¦ Less likely to miss tiny tumors due to high spatial res
  • ГўВќЕ’ Only 3-bit = 8 gray levels
  • ГўВќЕ’ False contouring Гўв‚¬вЂќ misses subtle intensity differences
  • ГўВќЕ’ Cannot distinguish similar tissue types ГўвЂ вЂ™ wrong diagnosis
Prototype 2 Low Spatial, High Intensity
  • ГўВќЕ’ Lower spatial resolution ГўвЂ вЂ™ coarser image
  • ГўВќЕ’ More likely to miss tiny tumors (spatial detail lost)
  • ГўЕ“вЂ¦ 12-bit = 4096 gray levels
  • ГўЕ“вЂ¦ Excellent tissue contrast discrimination
  • ГўЕ“вЂ¦ Less false contouring

Г°ЕёвЂ™ВЎ Which to Improve?

For detecting tiny tumors in MRI, improving intensity resolution (from 3-bit to higher) is the better investment. Tumors often have similar physical size but differ subtly in tissue density. The ability to distinguish these subtle intensity differences (contrast) is what makes MRI diagnostically useful. Spatial resolution matters for location, but contrast resolution determines if you can see the tumor at all.

3.2 Types of Adjacency

Let V be the set of intensity values used to define adjacency. Two pixels are adjacent if they are neighbors and both have values in V.

4-Adjacency

Pixels p and q are 4-adjacent if q ГўЛ†Л† N4(p) AND both have values from V.

$$p \sim_4 q \iff q \in N_4(p) \text{ and } f(q) \in V$$
8-Adjacency

Pixels p and q are 8-adjacent if q ГўЛ†Л† N8(p) AND both have values from V. (More permissive Гўв‚¬вЂќ allows diagonal connections.)

$$p \sim_8 q \iff q \in N_8(p) \text{ and } f(q) \in V$$
m-Adjacency (Mixed) Most important!

Pixels p and q are m-adjacent if either:

  1. q ГўЛ†Л† N4(p) Гўв‚¬вЂќ they are 4-adjacent, OR
  2. q ГўЛ†Л† ND(p) AND N4(p) ГўЛ†В© N4(q) has no pixels with values in V
$$p \sim_m q \iff \begin{cases} q \in N_4(p), & \text{OR} \\ q \in N_D(p) \text{ and } N_4(p) \cap N_4(q) \cap V = \emptyset \end{cases}$$

Why m-adjacency?

8-adjacency can create ambiguous paths Гўв‚¬вЂќ multiple paths between two pixels. m-adjacency eliminates this ambiguity by preferring 4-neighbors when possible. It only uses diagonal connections when the 4-neighbor path does NOT already pass through a V-valued pixel.

4.2 Activation Functions

Sigmoid / Logistic Used in Hidden Layer 1
$$\sigma(z) = \frac{1}{1 + e^{-z}} \quad \in (0, 1)$$
  • ГўЕ“вЂ¦ Output always between 0 and 1 Гўв‚¬вЂќ good for probability
  • ГўЕ“вЂ¦ Smooth, differentiable everywhere
  • ГўВќЕ’ Vanishing gradient for very large/small inputs (saturates)
  • Г°ЕёвЂњЕ’ Used in output for binary classification
ReLU (Rectified Linear Unit) Used in Hidden Layer 2
$$\text{ReLU}(z) = \max(0, z) = \begin{cases} z & \text{if } z \geq 0 \\ 0 & \text{if } z < 0 \end{cases}$$
  • ГўЕ“вЂ¦ No vanishing gradient for positive values
  • ГўЕ“вЂ¦ Computationally cheap
  • ГўЕ“вЂ¦ Most widely used in deep/CNN models
  • ГўВќЕ’ "Dying ReLU" Гўв‚¬вЂќ neurons can get stuck at 0
Softmax Suitable for multi-class output
$$\text{Softmax}(z_k) = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}$$
  • ГўЕ“вЂ¦ Outputs sum to 1 Гўв‚¬вЂќ interpretable as class probabilities
  • ГўЕ“вЂ¦ Ideal for multi-class classification (K > 2 classes)
  • ГўЕ“вЂ¦ Unlike other activation functions, Softmax operates on ALL output neurons simultaneously
  • Г°ЕёвЂњЕ’ Q3 Answer: Softmax differs from Sigmoid/ReLU/Tanh because it considers ALL outputs together to produce a probability distribution. Other functions operate independently on each neuron.
Tanh
$$\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}} \quad \in (-1, 1)$$
  • ГўЕ“вЂ¦ Zero-centered (unlike Sigmoid)
  • ГўЕ“вЂ¦ Range (ГўЛ†вЂ™1, 1) Гўв‚¬вЂќ useful when negative outputs matter
  • ГўВќЕ’ Still suffers from vanishing gradient at extremes

4.3 Forward Propagation Algorithm

Step 1 Гўв‚¬вЂќ Initialize

Set input activations: a = [xГўвЂљВЃ, xГўвЂљвЂљ, ..., xГўвЂљв„ў]

Step 2 Гўв‚¬вЂќ For each layer l

$$z^{[l]} = W^{[l]} \cdot a^{[l-1]} + b^{[l]}$$

Step 3 Гўв‚¬вЂќ Apply activation

$$a^{[l]} = g^{[l]}(z^{[l]})$$

Step 4 Гўв‚¬вЂќ Output is final layer's activation

Г…В· = a[L] (predicted output)

Step 5 Гўв‚¬вЂќ Compute Loss

$$\mathcal{L} = -\sum_i \left[ y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i) \right]$$

4.4 Loss Functions

Binary Cross-Entropy (Log Loss)

$$\mathcal{L}_i = -y_i \log(o_i) - (1-y_i)\log(1-o_i)$$

Per output neuron. y = actual, o = predicted.

Categorical Cross-Entropy

$$\mathcal{L} = -\sum_{k} y_k \log(\hat{y}_k)$$

Used with Softmax for multi-class output.

Q1 [6 marks] Гўв‚¬вЂќ Medical Imaging: Spatial vs Intensity Resolution

Prototype 1: Very high spatial res, only 3-bit per pixel (8 gray levels)
Prototype 2: Lower spatial res, 12-bit per pixel (4096 gray levels)

(i) Which is more likely to miss detecting tiny tumors? [2]

Answer: Prototype 2 (lower spatial resolution). Tiny tumors occupy very few pixels. With coarser spatial resolution, the tumor may span less than 1 pixel and become invisible Гўв‚¬вЂќ it literally gets averaged away. Spatial resolution determines the minimum detectable feature size.

Г°ЕёвЂ™ВЎ Key insight: High spatial res = more pixels per area = finer detail captured = tiny structures visible.

(ii) Which causes incorrect diagnosis due to loss of tissue contrast? [2]

Answer: Prototype 1 (3-bit = only 8 gray levels). MRI distinguishes tissue types by subtle differences in signal intensity. With only 8 levels, many different tissues get mapped to the same gray value Гўв‚¬вЂќ false contouring Гўв‚¬вЂќ making it impossible to distinguish healthy from cancerous tissue. This directly causes misdiagnosis.

$$\text{3-bit} \Rightarrow L=2^3=8 \text{ levels}\quad \text{vs} \quad \text{12-bit} \Rightarrow L=2^{12}=4096 \text{ levels}$$

(iii) Which parameter would you improve? [2]

Answer: Improve Intensity Resolution (increase bits per pixel).
Justification: For tumor detection in MRI, the core challenge is distinguishing tumor tissue from surrounding healthy tissue Гўв‚¬вЂќ they often have nearly identical physical sizes but differ in signal density. 3-bit depth (8 levels) is catastrophically inadequate for medical use. Going to even 8-bit (256 levels) dramatically improves diagnostic accuracy. Spatial resolution can compensate somewhat via zoom/magnification post-processing, but you cannot recover lost intensity information once quantized to 8 levels.

Formula: Log Loss = ГўЛ†вЂ™yГЎВµВўГ‚В·log(o_outГЎВµВў) ГўЛ†вЂ™ (1ГўЛ†вЂ™yГЎВµВў)Г‚В·log(1ГўЛ†вЂ™o_outГЎВµВў)

Loss for Output 2 (yГўвЂљвЂљ=0.0, oГўвЂљвЂљ=0.5474):

(ii) Output activation ГўвЂ вЂ™ Softmax

Practice P5 Comprehensive Image Concepts [6 marks]

(i) A grayscale image has 1024ГѓвЂ”768 resolution at 8-bit depth. Calculate the total number of pixels, gray levels, and file size in KB. [2]
(ii) The same image is converted to binary (1-bit). What visual artifact might appear on diagonal edges? [2]
(iii) In pixel adjacency, why is m-adjacency preferred over 8-adjacency for path tracing? [2]

ГўвЂ“В¶ Show Solution

(i) Image parameters

$$\text{Total pixels} = 1024 \times 768 = 786{,}432 \text{ pixels}$$ $$\text{Gray levels} = 2^8 = 256$$ $$\text{File size} = \frac{1024 \times 768 \times 8}{8 \times 1024} = \frac{786{,}432}{1024} = 768 \text{ KB}$$

(ii) 1-bit binary ГўвЂ вЂ™ Jagged Lines (Aliasing)

Diagonal edges that were smooth in 8-bit become jagged staircase patterns. With only 2 intensity levels (0 or 1), fine angular information is lost Гўв‚¬вЂќ pixels can only be fully ON or OFF, so curves and diagonals appear as staircases. This is spatial aliasing from extreme quantization.

(iii) m-adjacency vs 8-adjacency for path tracing

8-adjacency allows multiple paths between two pixels (via both N4 and ND neighbors), creating path ambiguity. Two pixels may have more than one shortest path, making algorithms like flood-fill or segmentation non-deterministic. m-adjacency eliminates this by using diagonal connections ONLY when no 4-neighbor alternative exists, guaranteeing a unique, unambiguous path between connected pixels.

ГўЕЎВЎ Quick Cheatsheet Гўв‚¬вЂќ CT Formula Sheet

Image Fundamentals

  • Gray Levels: $L = 2^k$
  • Storage: $\frac{M \times N \times k}{8}$ bytes
  • Total Pixels: $M \times N$

Activation Functions

  • Sigmoid: $\frac{1}{1+e^{-z}}$, range (0,1)
