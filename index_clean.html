Introduction to Digital Images

From analog signals to discrete pixel arrays

1.1 What is a Digital Image?

A digital image is represented as a 2D function f(x, y) where x and y are spatial coordinates and f is the intensity (gray level) at that point. Both coordinates and amplitude are finite, discrete quantities.

$$f(x,y) \quad \text{where} \quad x \in [0, M-1],\ y \in [0, N-1]$$

M = number of rows, N = number of columns

Grayscale Image

Each pixel has a single intensity value (typically 0ΓÇô255 for 8-bit).

$$f = \begin{bmatrix} 10 & 80 & 180 \\ 50 & 120 & 200 \\ 30 & 90 & 255 \end{bmatrix}$$

3├ù3 grayscale image matrix

RGB Color Image

Each pixel has 3 channels ΓÇö Red, Green, Blue ΓÇö each 0ΓÇô255.

$$f(x,y) = \begin{pmatrix} R \\ G \\ B \end{pmatrix},\quad \text{e.g.} \begin{pmatrix} 255 \\ 128 \\ 0 \end{pmatrix}$$

24-bit color = 8 bits ├ù 3 channels

1.2 Pixel Depth (Bit Depth)

The number of bits per pixel determines how many intensity levels can be represented. The formula: Intensity Levels = 2k where k = bits per pixel.

$$\text{Intensity Levels} = 2^k \quad \Rightarrow \quad L = 2^k$$ $$\text{e.g., } k=8 \Rightarrow L = 256 \text{ levels (0 to 255)}$$

Sampling, Quantization & Resolution

The mathematics of continuous-to-discrete conversion

2.1 Spatial Resolution

Spatial resolution defines how many pixels per unit area. It is measured in DPI (Dots Per Inch) or PPI (Pixels Per Inch). Higher DPI = more pixels = more detail captured.

$$\text{Total Pixels} = M \times N$$ $$\text{Spatial Resolution (DPI)} = \frac{\text{pixels}}{\text{inch}}$$

Effect of Reducing Spatial Resolution

  • Γ¼ç∩╕Å Fewer pixels ΓåÆ Jagged lines (staircase effect)
  • Γ¼ç∩╕Å Fine details (tiny tumors) become invisible
  • Γ¼ç∩╕Å Objects lose shape fidelity
  • Γ£à Smaller file size

Jagged Lines (Aliasing)

When spatial resolution is too low, smooth diagonal edges appear as a "staircase" pattern because a coarser grid cannot represent fine angular detail.

2.2 Intensity Resolution

Intensity resolution defines how many distinct gray levels can be represented per pixel. It is determined by the number of bits per pixel (bit depth).

$$L = 2^k \quad \Rightarrow \quad \text{gray levels from } 0 \text{ to } L-1$$ $$\text{Intensity range: } [0,\ 2^k - 1]$$

ΓÜá∩╕Å False Contouring

When intensity resolution is too low (e.g., 2-bit = 4 levels), smooth gradients appear as abrupt bands of constant color ΓÇö called false contours. This happens because nearby tones that should be slightly different are mapped to the same discrete level.

Example: An X-ray with only 3 bits (8 levels) cannot distinguish subtle density changes in tissue, leading to incorrect diagnosis due to loss of tissue contrast.

Bits/pixelGray LevelsTissue ContrastProblem
3-bit 8 Very Poor False contouring, missed soft tissue differences
8-bit 256 Good Acceptable for most uses
12-bit 4096 Excellent Ideal for medical imaging (MRI, CT)

2.3 Spatial vs. Intensity Resolution Tradeoff

Prototype 1 High Spatial, Low Intensity
  • Γ£à Captures fine spatial detail (edges, small structures)
  • Γ£à Less likely to miss tiny tumors due to high spatial res
  • Γ¥î Only 3-bit = 8 gray levels
  • Γ¥î False contouring ΓÇö misses subtle intensity differences
  • Γ¥î Cannot distinguish similar tissue types ΓåÆ wrong diagnosis
Prototype 2 Low Spatial, High Intensity
  • Γ¥î Lower spatial resolution ΓåÆ coarser image
  • Γ¥î More likely to miss tiny tumors (spatial detail lost)
  • Γ£à 12-bit = 4096 gray levels
  • Γ£à Excellent tissue contrast discrimination
  • Γ£à Less false contouring

≡ƒÆí Which to Improve?

For detecting tiny tumors in MRI, improving intensity resolution (from 3-bit to higher) is the better investment. Tumors often have similar physical size but differ subtly in tissue density. The ability to distinguish these subtle intensity differences (contrast) is what makes MRI diagnostically useful. Spatial resolution matters for location, but contrast resolution determines if you can see the tumor at all.

2.4 Solved Example ΓÇö Sampling Parameters

Problem:

An image has M=512, N=512 with 8 bits per pixel. Calculate: (a) total gray levels, (b) storage in bytes, (c) new storage if reduced to 4-bit.

Step 1 ΓÇö Gray Levels

$$L = 2^k = 2^8 = 256 \text{ gray levels}$$

Step 2 ΓÇö Storage at 8-bit

$$\text{Storage} = \frac{512 \times 512 \times 8}{8} = 512 \times 512 = 262{,}144 \text{ bytes} \approx 256 \text{ KB}$$

Step 3 ΓÇö Storage at 4-bit (halved bit depth)

$$\text{Storage} = \frac{512 \times 512 \times 4}{8} = 131{,}072 \text{ bytes} \approx 128 \text{ KB}$$

Gray levels drop from 256 ΓåÆ 16. File size halves but false contouring may appear!

3.2 Types of Adjacency

Let V be the set of intensity values used to define adjacency. Two pixels are adjacent if they are neighbors and both have values in V.

4-Adjacency

Pixels p and q are 4-adjacent if q Γêê N4(p) AND both have values from V.

$$p \sim_4 q \iff q \in N_4(p) \text{ and } f(q) \in V$$
8-Adjacency

Pixels p and q are 8-adjacent if q Γêê N8(p) AND both have values from V. (More permissive ΓÇö allows diagonal connections.)

$$p \sim_8 q \iff q \in N_8(p) \text{ and } f(q) \in V$$
m-Adjacency (Mixed) Most important!

Pixels p and q are m-adjacent if either:

  1. q Γêê N4(p) ΓÇö they are 4-adjacent, OR
  2. q Γêê ND(p) AND N4(p) Γê⌐ N4(q) has no pixels with values in V
$$p \sim_m q \iff \begin{cases} q \in N_4(p), & \text{OR} \\ q \in N_D(p) \text{ and } N_4(p) \cap N_4(q) \cap V = \emptyset \end{cases}$$

Why m-adjacency?

8-adjacency can create ambiguous paths ΓÇö multiple paths between two pixels. m-adjacency eliminates this ambiguity by preferring 4-neighbors when possible. It only uses diagonal connections when the 4-neighbor path does NOT already pass through a V-valued pixel.

4.2 Activation Functions

Sigmoid / Logistic Used in Hidden Layer 1
$$\sigma(z) = \frac{1}{1 + e^{-z}} \quad \in (0, 1)$$
  • Γ£à Output always between 0 and 1 ΓÇö good for probability
  • Γ£à Smooth, differentiable everywhere
  • Γ¥î Vanishing gradient for very large/small inputs (saturates)
  • ≡ƒôî Used in output for binary classification
ReLU (Rectified Linear Unit) Used in Hidden Layer 2
$$\text{ReLU}(z) = \max(0, z) = \begin{cases} z & \text{if } z \geq 0 \\ 0 & \text{if } z < 0 \end{cases}$$
  • Γ£à No vanishing gradient for positive values
  • Γ£à Computationally cheap
  • Γ£à Most widely used in deep/CNN models
  • Γ¥î "Dying ReLU" ΓÇö neurons can get stuck at 0
Softmax Suitable for multi-class output
$$\text{Softmax}(z_k) = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}$$
  • Γ£à Outputs sum to 1 ΓÇö interpretable as class probabilities
  • Γ£à Ideal for multi-class classification (K > 2 classes)
  • Γ£à Unlike other activation functions, Softmax operates on ALL output neurons simultaneously
  • ≡ƒôî Q3 Answer: Softmax differs from Sigmoid/ReLU/Tanh because it considers ALL outputs together to produce a probability distribution. Other functions operate independently on each neuron.
Tanh
$$\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}} \quad \in (-1, 1)$$
  • Γ£à Zero-centered (unlike Sigmoid)
  • Γ£à Range (ΓêÆ1, 1) ΓÇö useful when negative outputs matter
  • Γ¥î Still suffers from vanishing gradient at extremes

4.3 Forward Propagation Algorithm

Step 1 ΓÇö Initialize

Set input activations: a = [xΓéü, xΓéé, ..., xΓéÖ]

Step 2 ΓÇö For each layer l

$$z^{[l]} = W^{[l]} \cdot a^{[l-1]} + b^{[l]}$$

Step 3 ΓÇö Apply activation

$$a^{[l]} = g^{[l]}(z^{[l]})$$

Step 4 ΓÇö Output is final layer's activation

┼╖ = a[L] (predicted output)

Step 5 ΓÇö Compute Loss

$$\mathcal{L} = -\sum_i \left[ y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i) \right]$$

4.4 Loss Functions

Binary Cross-Entropy (Log Loss)

$$\mathcal{L}_i = -y_i \log(o_i) - (1-y_i)\log(1-o_i)$$

Per output neuron. y = actual, o = predicted.

Categorical Cross-Entropy

$$\mathcal{L} = -\sum_{k} y_k \log(\hat{y}_k)$$

Used with Softmax for multi-class output.

Q1 [6 marks] ΓÇö Medical Imaging: Spatial vs Intensity Resolution

Prototype 1: Very high spatial res, only 3-bit per pixel (8 gray levels)
Prototype 2: Lower spatial res, 12-bit per pixel (4096 gray levels)

(i) Which is more likely to miss detecting tiny tumors? [2]

Answer: Prototype 2 (lower spatial resolution). Tiny tumors occupy very few pixels. With coarser spatial resolution, the tumor may span less than 1 pixel and become invisible ΓÇö it literally gets averaged away. Spatial resolution determines the minimum detectable feature size.

≡ƒÆí Key insight: High spatial res = more pixels per area = finer detail captured = tiny structures visible.

(ii) Which causes incorrect diagnosis due to loss of tissue contrast? [2]

Answer: Prototype 1 (3-bit = only 8 gray levels). MRI distinguishes tissue types by subtle differences in signal intensity. With only 8 levels, many different tissues get mapped to the same gray value ΓÇö false contouring ΓÇö making it impossible to distinguish healthy from cancerous tissue. This directly causes misdiagnosis.

$$\text{3-bit} \Rightarrow L=2^3=8 \text{ levels}\quad \text{vs} \quad \text{12-bit} \Rightarrow L=2^{12}=4096 \text{ levels}$$

(iii) Which parameter would you improve? [2]

Answer: Improve Intensity Resolution (increase bits per pixel).
Justification: For tumor detection in MRI, the core challenge is distinguishing tumor tissue from surrounding healthy tissue ΓÇö they often have nearly identical physical sizes but differ in signal density. 3-bit depth (8 levels) is catastrophically inadequate for medical use. Going to even 8-bit (256 levels) dramatically improves diagnostic accuracy. Spatial resolution can compensate somewhat via zoom/magnification post-processing, but you cannot recover lost intensity information once quantized to 8 levels.

Q2 [2 marks] ΓÇö 16├ù16 Image ΓåÆ ANN Design

Dataset: 16├ù16 pixel images where each pixel is a feature. Goal: classify into 5 categories.

Input neurons:

$$\text{Input neurons} = 16 \times 16 = 256$$

Each pixel is one feature ΓåÆ one input neuron per pixel.

Output neurons:

$$\text{Output neurons} = 5 \quad (\text{one per class})$$

Activation function for output:

Softmax ΓÇö because this is a multi-class classification problem (5 classes). Softmax converts the raw output scores into a probability distribution over 5 classes that sums to 1. We pick the class with highest probability.

$$\text{Softmax}(z_k) = \frac{e^{z_k}}{\sum_{j=1}^{5} e^{z_j}}$$

A company is developing a satellite imaging system. Prototype A captures 4096├ù4096 pixel images with only 2-bit intensity. Prototype B captures 1024├ù1024 images with 16-bit intensity.
(i) Which prototype is better for detecting thin roads? [2]
(ii) Which is better for classifying land-use types (forest vs urban vs water)? [2]
(iii) What artifact will appear in Prototype A's images? Explain. [2]

Γû╢ Show Solution

(i) Thin roads ΓåÆ Prototype A

Roads are narrow structures requiring fine spatial detail. Prototype A's 4096├ù4096 spatial resolution captures sub-meter detail. Prototype B's 1024├ù1024 may cause thin roads to disappear.

(ii) Land-use classification ΓåÆ Prototype B

Forest, urban, and water differ in spectral reflectance intensity. 16-bit = 65,536 levels provides excellent discrimination. 2-bit = only 4 levels ΓÇö forest and water may map to the same level!

(iii) Artifact in Prototype A ΓåÆ False Contouring

$$2\text{-bit} \Rightarrow L = 2^2 = 4 \text{ gray levels only}$$

With only 4 levels, smooth intensity gradients (e.g., sunrise lighting on terrain) will appear as abrupt bands ΓÇö false contours ΓÇö making natural features look artificially stepped.

Practice P3 ANN Design Question [4 marks]

You are building a medical image classification system for MRI scans. Each scan is 32├ù32 pixels. The system must classify into 3 disease categories: "Healthy", "Stage 1", "Stage 2".
(i) How many neurons in the input and output layer? [1]
(ii) Which activation for the output layer and why? [1]
(iii) A developer suggests using Sigmoid in the output instead. What problem would this cause? [2]

Γû╢ Show Solution

(i) Neuron count

$$\text{Input} = 32 \times 32 = 1024, \quad \text{Output} = 3$$

(ii) Output activation ΓåÆ Softmax

Softmax is correct because: (a) outputs sum to 1.0, interpretable as probabilities, (b) designed for mutually exclusive multi-class problems, (c) forces competition between classes.

(iii) Problem with Sigmoid on 3-class output

Sigmoid treats each output independently. The three outputs could all be 0.8, 0.9, 0.7 ΓÇö summing to 2.4, which is meaningless as a probability distribution. You cannot determine which class is "most likely" without additional normalization. The outputs don't compete with each other, so the model can simultaneously predict all classes as highly probable ΓÇö an incoherent result.

Practice P4 Mini Forward Propagation [6 marks]

Consider a simple ANN: Input [x1=0.5, x2=0.3], one hidden neuron h1 (Sigmoid), one output o1 (Sigmoid).
Weights: w(x1ΓåÆh1)=0.6, w(x2ΓåÆh1)=0.4, w(h1ΓåÆo1)=0.8. No biases.
Actual output: y=1.
(i) Calculate h1 output [2] (ii) Calculate o1 output [2] (iii) Calculate log loss [2]

Γû╢ Show Solution

(i) Hidden neuron h1

$$z_{h1} = (0.5)(0.6) + (0.3)(0.4) = 0.30 + 0.12 = 0.42$$ $$a_{h1} = \sigma(0.42) = \frac{1}{1+e^{-0.42}} = \frac{1}{1+0.6570} \approx \mathbf{0.6035}$$

(ii) Output neuron o1

$$z_{o1} = (0.6035)(0.8) = 0.4828$$ $$a_{o1} = \sigma(0.4828) = \frac{1}{1+e^{-0.4828}} \approx \frac{1}{1+0.6171} \approx \mathbf{0.6183}$$

(iii) Log Loss (y=1, ┼╖=0.6183)

$$\mathcal{L} = -(1.0)\ln(0.6183) - (0)\ln(1-0.6183)$$ $$= -(-0.4804) = \mathbf{0.4804}$$

Lower than CT Q4's loss because the prediction (0.618) is closer to the correct label (1) than in Q4 (0.453).

Practice P5 Comprehensive Image Concepts [6 marks]

(i) A grayscale image has 1024├ù768 resolution at 8-bit depth. Calculate the total number of pixels, gray levels, and file size in KB. [2]
(ii) The same image is converted to binary (1-bit). What visual artifact might appear on diagonal edges? [2]
(iii) In pixel adjacency, why is m-adjacency preferred over 8-adjacency for path tracing? [2]

Γû╢ Show Solution

(i) Image parameters

$$\text{Total pixels} = 1024 \times 768 = 786{,}432 \text{ pixels}$$ $$\text{Gray levels} = 2^8 = 256$$ $$\text{File size} = \frac{1024 \times 768 \times 8}{8 \times 1024} = \frac{786{,}432}{1024} = 768 \text{ KB}$$

(ii) 1-bit binary ΓåÆ Jagged Lines (Aliasing)

Diagonal edges that were smooth in 8-bit become jagged staircase patterns. With only 2 intensity levels (0 or 1), fine angular information is lost ΓÇö pixels can only be fully ON or OFF, so curves and diagonals appear as staircases. This is spatial aliasing from extreme quantization.

(iii) m-adjacency vs 8-adjacency for path tracing

8-adjacency allows multiple paths between two pixels (via both N4 and ND neighbors), creating path ambiguity. Two pixels may have more than one shortest path, making algorithms like flood-fill or segmentation non-deterministic. m-adjacency eliminates this by using diagonal connections ONLY when no 4-neighbor alternative exists, guaranteeing a unique, unambiguous path between connected pixels.

ΓÜí Quick Cheatsheet ΓÇö CT Formula Sheet

Image Fundamentals

  • Gray Levels: $L = 2^k$
  • Storage: $\frac{M \times N \times k}{8}$ bytes
  • Total Pixels: $M \times N$

Activation Functions

  • Sigmoid: $\frac{1}{1+e^{-z}}$, range (0,1)
  • ReLU: $\max(0,z)$, range $[0,\infty)$
  • Softmax: $\frac{e^{z_k}}{\sum e^{z_j}}$, sums to 1
  • Tanh: $\frac{e^z-e^{-z}}{e^z+e^{-z}}$, range (-1,1)

Adjacency Rules (V = intensity set)

  • 4-adj: q Γêê N4(p) and f(q) Γêê V
  • 8-adj: q Γêê N8(p) and f(q) Γêê V
  • m-adj: q Γêê N4(p) OR [q Γêê ND(p) and N4(p)Γê⌐N4(q)Γê⌐V=Γêà]

Loss & ANN

  • Forward: $z = \sum w_i a_i$, then $a = g(z)$
  • Log Loss: $-y\log(o) - (1-y)\log(1-o)$
  • Input nodes = image pixels (M├ùN)
  • Output nodes = number of classes
